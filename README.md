# Тестовое задание

Создать сайт с функцией сокращения URL.

---

## Требования

- **Платформа:** последняя доступная версия **.NET**
- **Функционал сокращения:** разработан самостоятельно, без готовых решений
- **Сокращённые ссылки:** создаются так, чтобы их нельзя было предугадать  
  *(нельзя использовать простой последовательный порядок: 1, 2, 3… или a, b, c…)*
- **Хранение данных:** база данных **MySQL** (MariaDB)
- **Миграции:** автоматические миграции и первоначальное создание БД
- **Запуск:** достаточно указать строку подключения и нажать запуск
- **Валидация:** проверки на некорректный ввод данных
- **Аналитика:** при переходе по короткой ссылке — подсчёт числа переходов

---

## Интерфейс

На главной странице после запуска — **таблица** со столбцами:

| Столбец           | Описание              |
|-------------------|------------------------|
| Длинный URL       | Исходная ссылка       |
| Сокращённый URL   | Сгенерированная ссылка|
| Дата создания     | Когда создана запись  |
| Количество переходов | Счётчик кликов    |
| Управление        | Кнопки действий       |

### Столбец «Управление»

Две кнопки:

1. **Редактировать**  
   Редактирование длинной ссылки (режим на усмотрение: инлайн, отдельная страница или попап).

2. **Удалить**  
   Удаление записи.

---

## Будет плюсом

- Тесты
- Комментарии в коде  
  *Комментарий должен отвечать на вопрос «Почему?», а не «Что?»*
- Оптимизация под нагрузку  
  *Представьте 100 / 1000 / 10 000 запросов в секунду*
- Простота решения  
  *Сайт должен выполнять поставленную задачу, а не быть «комбайном на все случаи жизни»*
